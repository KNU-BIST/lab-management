## How to log your work

### Starting a new experiment
1. Go to **Issues → New Issue**
2. Choose **Experiment log** template
3. Fill in the form — objective, method, expected date
4. Assign yourself and add the relevant label (e.g. `RNA-seq`, `immunology`)
5. The PI can see it immediately on the Projects board

### Logging a task
1. **Issues → New Issue → Task** template
2. Assign to yourself or a team member
3. Set a due date in the title or description

### Updating progress
- Open your experiment Issue and **leave a comment** with your update
- Check off boxes in the checklist as you complete steps
- When done: move the card to **Done** on the Projects board and close the Issue

### Adding a protocol to the Wiki
1. Go to **Wiki → New Page**
2. Use the **Protocol update** Issue template as your draft first
3. Once approved by PI, copy it into the Wiki

## Labels guide

| Label | Meaning |
|-------|---------|
| `experiment` | An active or completed experiment |
| `task` | An administrative or analytical task |
| `protocol` | A new or updated lab SOP |
| `needs-PI-review` | Add this when you need the PI's input or approval |
| `blocked` | Something is preventing progress — flag immediately |
| `RNA-seq` | Specific to RNA-seq / transcriptomics work |
| `immunology` | Immunology / inflammation experiments |
| `cancer` | Cancer biology experiments |
| `urgent` | Needs attention within 24 hours |
